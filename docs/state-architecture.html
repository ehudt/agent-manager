agent-manager — state detection & presentation

How a session's state is derived from process / hook / Claude-JSONL signals, and how that single derivation feeds every UI surface (CLI, tmux status bar, Go TUI).

1. The seven states

One enum drives every glyph, color, and badge in the tool. Declared in lib/state.sh; glyph + color table lives in lib/status-bar.

▸running
Agent producing output / executing a tool.
●waiting_input
Turn ended; prompt parked at ❯.
⚠waiting_permission
Claude/Codex y/n approval prompt.
◆waiting_custom
Elicitation dialog / plan / /ask.
◌starting
Session < 5 s old, agent not yet attached.
○idle
Shell prompt, agent exited cleanly.
✕dead
No tmux session or shell exit with errors.

2. Signal sources, ranked

Three independent signal sources answer "what is this session doing?". Lower-cost, higher-fidelity sources come first; later sources are fallbacks.

RANK 1 — push
Hook state file bash
Claude Code (and Codex) fires lifecycle hooks → lib/hooks/state-hook.sh writes one word to /tmp/am-state/<session>. Authoritative for waiting_* — those persist across days with no churn. running has a 180 s staleness gate (catches crashed-mid-tool).
RANK 2 — pull, structured
Claude JSONL tail bash
For Claude only. Last meaningful entry in ~/.claude/projects/<encoded-dir>/<sid>.jsonl tells us whether the assistant's stop_reason was end_turn (→ waiting_input) or tool_use (→ running). Session id resolved via sidecar → --session-id argv → lsof, never raw mtime guess.
RANK 3 — pull, fuzzy
tmux pane content bash
Capture last 40 lines, strip ANSI, regex-match: Claude/Codex permission prompts, plan dialogs, (running) tags, spinner, idle ❯. Used when hook/JSONL silent and for non-Claude agents that have no JSONL.
Process-level sanity check (runs first)
_state_pane_is_shell(session) — if the top pane PID is a shell with no agent child, the agent has exited. Branches to starting (age < 5 s) or agent_classify_exit → idle/dead.

3. The resolver — _state_resolve

Single canonical path. Called both per-session (agent_get_state) and bulk (lib/status-bar with nameref maps). Keeps the sidebar glyph and the CLI badge in lockstep.

STEP 1
shell check
Pane is bare shell? → starting / idle / dead. short-circuit.
STEP 2
hook terminal
/tmp/am-state/<s> says waiting_*? → return immediately, skip pane fork.
STEP 3
pane scan
Match permission / custom / dead / idle from pane content. Skip alive-check (done in step 1).
STEP 4
JSONL (claude)
Reverse-tail conversation JSONL. stop_reason → state. Targeted by sid, not mtime.
STEP 5
fallback
pane running / waiting_input, then hook running, then conservative running.

4. Hook flow — push path

Why push wins for waiting states: an idle session can sit at waiting_input for days without polling. Hook writes the word once; everyone reads forever.

Event → state mapping

Hook eventMatcherState written
Stop—waiting_input
Notificationidle_promptwaiting_input
Notificationpermission_promptwaiting_permission
Notificationelicitation_dialogwaiting_custom
PermissionRequest—waiting_permission
UserPromptSubmit—running
PreToolUse / PostToolUse—running

Race guard: running writes are dropped when the current file is waiting_*, unless the event is UserPromptSubmit (only explicit user action transitions waiting → running).

5. Presentation surfaces

Three consumers — all call the same _state_resolve (bash) or read the same hook/registry files (Go). Glyphs match across surfaces because they share lib/status-bar's GLYPH / COLOR tables.

CLI & orchestration bash

am peek, am send, agent-to-agent waits
  • agent_get_state <s> — single-shot resolver; one fork-per-call.
  • agent_wait_state <s> states timeout — polls 0.5 s, requires N stable polls + quiet activity before returning.
  • agent_classify_exit — pane scan after shell-prompt detected, picks idle vs dead.

tmux status bar bash

lib/status-bar — runs every 5 s + on hook refresh
  • Builds bulk fixtures: 1× tmux list-sessions, 1× list-panes, 1× ps, 1× jq — passes by nameref to _state_resolve.
  • Writes 3 options per session: @am_status_left (clickable tabs), @am_sidebar (pane border), @am_attention (counter).
  • N strips precomputed — switch is instant; tmux reads cached option, no shell fork.

Go TUI go

cmd/am-browse, cmd/am-list-internal
  • internal/sessions.LoadEntries — parses tmux + sessions.json directly.
  • ReapOrphans, RefreshTitles — mirror the bash registry_gc / auto_title_scan, share throttle markers in $AM_DIR.
  • Reads hook state files for glyphs; does not re-implement _state_resolve — relies on what status-bar wrote.

7. Key files

  • lib/hooks/state-hook.sh — event→state mapping, session identification, race guard.

  • internal/sessions/reap.go — Go mirror of registry_gc (orphan rows + hook state files).

  • cmd/am-list-internal/main.go — fast list generator for fallback fzf path.
