agent-manager · State Detection & Presentation v2 — hook-only resolver · lib/state.sh, lib/hooks/state-hook.sh, lib/status-bar, cmd/am-browse, internal/sessions/

Architecture change. Resolver previously merged four signal layers (hooks · pane scrape · Claude JSONL · ps tree). With Claude Code + Codex hooks both wired, pane regex and JSONL parsing were removed. Resolver collapsed to two sources: ps tree + hook file. A new unknown state replaces silent running when no signal is available.
state.sh: 740 → 300 lines resolver stages: 5 → 3 on-disk artifacts: 7 → 6 (.sid sidecar gone) state_lab cases: 12 → 3

State of every am session is resolved on-demand by a single canonical resolver (_state_resolve). Pipeline is now shell check → hook file → unknown. Two consumers render the result: the bash status-bar that paints tmux tabs, and the Go am-browse TUI that lists sessions for the picker (state shown only in the tab strip).

Signal sources

Hook files (push, primary)
lib/hooks/state-hook.sh

Claude Code and Codex fire lifecycle hooks. Script maps event → am state and writes one of waiting_input | waiting_permission | waiting_custom | running to $AM_STATE_DIR/<session>.

Session resolved by $AM_SESSION_NAME → tmux #S of $TMUX_PANE → registry cwd match. Race guard skips running writes when state file already shows a waiting_* unless event is UserPromptSubmit (only user action transitions waiting → running).

tmux + ps tree (pull)
ps + tmux display-message / list-panes

Top pane PID + descendants drive the shell-vs-agent distinction. A shell at the top of the pane means the agent has exited (idle or dead) or the session is brand-new (starting, age < 5s).

Removed (v1 layers)
pane scrape · Claude JSONL · .sid sidecar

ANSI-stripped pane regex (Codex spinner, Claude /ask, plan approval); JSONL tail parse for stop_reason; per-session conversation UUID cache.

Coverage moved to hooks: Codex UserPromptSubmit/PreToolUse/PostToolUse/PermissionRequest/Stop + Claude Notification[idle_prompt|permission_prompt|elicitation_dialog] are the equivalent signals.

Cache & coordination files

$AM_STATE_DIR/<session>
One-line state file written by state-hook.sh. Read by _state_hook_read. Removed by ReapOrphans.
$AM_DIR/sessions.json
Registry. Holds directory / branch / agent_type / task per session. Read by both bash and Go (ReadRegistry); atomic temp-rename writes.
$AM_DIR/.gc_last
60s throttle shared by bash registry_gc and Go ReapOrphans. Whichever runs first stamps the marker.
$AM_DIR/.title_scan_last
60s throttle shared by bash auto_title_scan and Go RefreshTitles. Hooks invalidate it on prompt boundaries so the next status-bar tick refreshes within ~5s.
$AM_DIR/.list_cache
Cached am list output. Hooks delete it so the next fzf reload picks up the new state.
$AM_DIR/.state-debug.log
Opt-in trace: AM_STATE_DEBUG=1 writes one line per resolution showing which step won (shell | classify_exit | hook | fallback).
$AM_DIR/.hook-debug.log
Opt-in trace: AM_HOOK_DEBUG=1 writes one line when a hook fires but exits without writing state (registry miss, no session resolvable, cwd inaccessible).

Invariants worth remembering

  • Single resolver. Both agent_get_state (per-session) and status-bar (bulk tick) call _state_resolve. Bulk fixtures are passed by bash 4.3+ nameref; without them the resolver builds size-1 fixtures inline.
  • Hook is authoritative. No pane scrape, no JSONL parse. Hooks fire from Claude Code + Codex; absence of a hook signal is reported as unknown, not invented.
  • 180s running gate. A stale running in the hook file falls through to unknown. Catches the "agent crashed mid-tool-call" case where no terminal PostToolUse / Stop arrives.
  • Race guard. state-hook.sh refuses to overwrite a waiting_* state with a delayed running event. Only UserPromptSubmit may transition waiting → running.
  • Status-bar pre-renders one strip per perspective. Every session's @am_status_left already has itself marked current — tmux switch reads the precomputed option.
  • Throttles are cross-language. .gc_last / .title_scan_last coordinate bash and Go without IPC.
  • Browser presents tasks, not state. The Go TUI shows registry task; live am-state appears only via the tmux tab strip painted by status-bar.
